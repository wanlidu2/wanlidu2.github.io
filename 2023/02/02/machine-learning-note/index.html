Overview of machine learning
Machine learning is started from the spam filter well it is realized with the machine learning principles. It is a science that means computers can learn from data by programming. Usually, computers can learn from several loops while learning from data, they can execute by the written rules and then evaluate the target value and print out the good results or improve the mechanism by fixing the analysis errors. Data mining is used to discover patterns in digging large amounts of data.
Machine learning is commonly used to solve problems with existing solutions but also requires hand-tuning or long lists of rules, currently not well-solved complex problems, fluctuating environments with fresh data updated, and targets with a large amount of data.
We can classify machine learning systems by analyzing whether the data are trained with human supervision, whether they can learn incrementally on the fly, and whether they compare the new data or build a predictive model.
In supervised learning, the training data typed into the algorithm includes the desired solutions called labels. For example, the classification progress will let the system learn from the class, the predictors can analyze a set of features and then get a useful regression. K-Nearest Neighbors, Linear Regression, Logistic Regression, Support Vector Machines, Decision Trees and Random Forests, and Neural networks are some algorithms applied to supervised learning methods. Other learning algorithms are unsupervised learning, semisupervised learning, and reinforcement learning.
The system incapable of incremental learning is called batch learning while it must be trained using all the available data. This method is not convenient when solving large amounts of data. By contrast, in online learning, the system can learn about new data on the fly.
When constructing an algorithm for some specific question, we may face a lot of problems. Therefore, we always need to be aware of the insufficient quantity of training data, nonrepresentative training data, and poor quality data. We can choose to drop some irrelevant features and avoid fitting problems in the training data. If we focus too much and overgeneralize the training data, it will cause overfitting. The possible solutions are simplifying the model by selecting fewer parameters, gathering more training data, and reducing the noise in the training data. The underfitting model is too simple to learn the underlying structure of the data, so we need to add more parameters to the model and feed better features to predict. The system will not perform well if the training data is not enough or does not represent, noisy, or polluted.
The process to check the model’s performance in new data can be realized by splitting the original data into the training set and the test set. If we want to select better algorithms, we can import hyperparameter methods to check their performance.

A complete project process
The main steps are: have a look at the big picture of the whole dataset, import the data and packages, discover and visualize the data to gain insights, prepare the data for machine learning algorithms, select a model and train it, fine-tune the parameters and settings in the model, present the solution, then launch, monitor, and maintain the system.
Take the California Housing Price dataset for example.
Look at the big picture: We find out the implements of the data.
Frame the problem: Clarify the problem we need to solve. If the processing is complex, we can use pipelines to integrate some methods.
Select a performance measure: If we want to predict a numerical feature, we can evaluate the performance by calculating the errors between the predicted value and the real value. For example, we can compare Root Mean Square Error (RMSE), Mean Absolute Error (MSE), or other methods.
Check the assumptions: One of the good ideas is to list some assumptions and then verify them.
Create a workspace and download the data
Create a test set: Pick some instances randomly for example 20% of the dataset. Commonly we can use the train_test_split command from the sklearn’s model_selection
and set the test ratio.
Discover and visualize the data to gain insights: Besides the training set, the other remaining part can also be explored. In case not changing the original data, we can create a copy of the dataset.
Visualize Geographical data: For geographical information.
Look for correlations: We can compute the standard correlation coefficient by using the corr() method. Besides, we can subtract some relationships by drawing the scatter matrix of the features by finding whether there exist some downward or upward slopes.
Experiments with attribute combinations: After finding out some correlations we can construct new features for the dataset and check their performance.
Prepare the data for Machine Learning algorithms: We can use the transformation more easily on any dataset.
Data cleaning: First, we need to have a cleaning training dataset by dropping the target values, we can choose to get rid of the corresponding districts dropna(), the whole attributes drop(), or set values by fillna(). In sklearn, the simple imputer provides such as median methods to fill the null value in some numerical features.
Handling text and categorical attributes: For those text features, we can encode them with OrdinalEncoder in sklearn.processing and draw OneHotEncoder to present the relationships among features and the target value.
Custom transformers: Besides the Scikit-Learn, we can specifically own transformers by defining new classes and functions according to the target dataset.
Feature Scaling: We can use min-max scaling or standardization to control the scale of the attributes. MinMaxScaler is a transformer in Scikit-Learn.
Transformation pipelines: The pipeline constructor takes a list of name/estimator pairs defining a sequence of steps. It expresses the same methods as the final estimators. Scikit-Learn provides ColumnTransformer to use a single transformer to handle all columns.
Select and Train a model: To begin with, we need to train and evaluate the training set and linear regression is a simple way to start with. Then we can improve the evaluation by using cross-validation. Instead of using train_test_split, we can take Scikit-Learn’s K-fold cross-validation feature. The random forest regressor is a kind of building a model on top of many other models, ensemble learning.
Fine-tune the model: Grid search is a way to fiddle with the hyperparameters manually, usually trying out consecutive values of the parameter. Another method is a randomized search which can select a random value for each hyperparameter at every iteration. If we want to combine multiple models to perform better we can use ensemble methods.
Analyze the best models and their errors: The RandomForestRegressor can indicate the relative importance of each attribute for making accurate predictions. We can call features_importances to compare their importance scores.
Evaluate the system on the test set: After getting the predictors and the labels from the test set, we run the full_pipeline to transform the data and then evaluate the model by the scores or the errors we set before.
