-1 Visualizing Data with ggplot2
Ggplot stands for Grammar of Graphics. It separates what is graphed from how it is graphed. The structure of ggplot2 commands can help you address the layered grammar of graphics. It is composed of geom_function (geometric), coordinate_function, and facet_function.
In geom_function part, it contains aes part, which stands for aesthetic elements, stat, and position.
Another form is qplot, which commands for quicker work but with less power and control. If you do not need to modify something, it is a clever idea to use that command.
More information can be found in Ggplot.tidyverse.org, exts.ggplot.tidyverse.org/gallery, RStudio also provides a cheat sheet for beginners to learn the functions of R.
Setting colors in R:
After loading data, we can set the color we want when we draw plots of the data.
By doing the question mark in console in R, we can get a brief introduction of colors provided by R.
If we text colors() we will get list of color names. What’s more, datalac.cc/rcolors shows the colors in a more intuitive way. If we want to enter the name of a color, we can put it into quotes in the barplot command. RGB triplets (0.00-1.00), (0-255), RGB hexcodes, index numbers, either of those ways works to set the color of plots and can turn those pictures more intuitive and interesting.
E.g. barplot(x,col=c colors() [589])
Using color palettes:
Palette() can show us the current colors in R and we can use the current palette by setting col= (valid numbers).
More examples can be found at colorbrewer2.org. We can enter p_load(RColorBrewer). These are extremely useful if we want to show diverging, sequential, or qualitative palettes.
Other palettes such as wesanerson, viridis color palettes, scientific jiournal, and Sci-Fi themed color palettes.
If you want to specify your own palettes, you can create a list and construct the color names in it.
Creating bar charts:
“Tabular format” is simply information presented in the form of a table with rows and columns. Before solving questions with data, we need to convert tables to rows and save this form of the data frame.
1. df <- data_name %>% enter the tabular data
2. as_tibble() %>% convert to tibble with rows
3. uncount(n) %>% convert from summary values
4. mutate_all(as_factor) %>% convert all vars to factors
5. mutate_all(fct_infreq) %>% order by descending factors
6. print() show the results
At last, we can use ggsave() as png and set the width, height, dpi.
More kinds of charts can be realized such as side-by-side bar chart, stacked bar chart, 100% stacked bar chart.

Box plots are commonly used to simplify the display of quantitative or measured variables.
Creating scatterplots:
When we prefer to have a look at the associations between quantitative variables.
Using geom_jitter() command can let the chart separate the data points so it helps you to see the overall pattern in a more intuitive way.
If we set size=X.Length, we will get a better comprehension of the association between things.
After geom_smooth(method=lm), we generate a linear regression line in the chart.
geom_density2d() can show the density of the data points.

-2 Wrangling Data
The form of tidy data:
I. column == variable/field/attribute
II. row == case/observation ( no headers or spaces
III. cell == single value encoded with text or numbers
IV. file == one level of observation/abstraction
In untidy data:
Spreadsheets would have some problems, we will meet formulas or refer back to things or some data saved in PDF form. The time series data (data spread in two different directions), XML/JSON data, and data with compound values are also untidy data.
We can turn untidy data into tidy data by generating a new data frame named tidy_ts, meaning to save it as a tsymbol or as a tsibble(time series tibble) then select variables with mutate().
After changing into tidy data, we can easily have further analysis.
Tibbles:
It is a way of storing data and viewing them in a variety of tables. Tibble has an enhanced version of a data frame. Tibbles is a great way of printing and displaying large datasets.
Use class(df) to get the class of df.
Use mutate( Tree= ) to convert it to factors
Using data.table:
Data table is used to work with a large load of data sets.
pacman::p_load(pacman, data.table, httr, tidyverse)

pacman: for loading/unloading packages

data.table: for working with large datasets

httr: for working with web sources

We can filter rows, select columns, or work out both tasks.
Converting the shape of data:
Using the function spread(key=, value= ) to category and split into columns, and set values for each category. This turn tall value to wide.
Function gathers to change the shape of data from wide to tall.
Converting data from tables to rows:
We can turn some compact formats that list categorical variables and combinations to be readable.
After entering the value into a data frame, we can text as_tibble() to convert it to tibble with rows.
Data with times:
If we need to deal with data describing dates, months, years, or many kinds of procedures, we can get help from some packages in R.
