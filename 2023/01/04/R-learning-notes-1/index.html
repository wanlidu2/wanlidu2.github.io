R learning notes 1
Category: R
Text:
R is a useful tool for data analysis. It is common when handling not only machine learning but also in academic fields, such as biology, astronomy in social science.
-1 General process
The typical analysis process is that
• Install and load packages: load the base packages, we can use ‘install.packages(‘’)’ or library(‘’) to achieve this.
• Load and prepare data: if we want to save the dataset to tibble (a package in R that is used to manipulate and print data frames) named “df” (data frame), ‘df <- ’. Besides, we have access to choose a part of the dataset to save and check the level of some specific column. mutate() adds new variables and preserves existing ones; transmute() adds new variables and drops existing ones.
• Show the outcome of variables: draw a bar chart of some characters, display frequency, filter out the NA elements, and charts of more than one element, and even draw metrics or divided the data by groups.
• Clean up: rm(list = ls()) to remove all objects from environment; p_unload(all) to remove all the packages; graphics.off() to clear all the plots and graphic devices; cat(“\014”) to clear the console window.
-2 Enter data:
Basic commands can solve the problem of basic math, and print number sequences or characters in the console. If we want to assign values to variables we should use <- instead of =. Multiple assignments work as well. The command c() means combining multiple values in a single variable. If we enter only variables, the contents will be printed in the console.
Numeric values are by default double precision. R does not distinguish a single character or collection of characters. Sequential data and metrics are supportable in R. However, the vector is set by default which does not have dimension.
Data types can be numeric, character, logical, and data structures like vectors, matrices, arrays, data frames, and lists. The List can be self-referential.
-3 Comments and headers:
With the sign #, we can temporarily turn off and on the code.
For example: if we want to draw a histogram, we can set the color, border, title, etc. With # temporarily appearing in the command, after re-running the whole command, it will change to the default settings or nothing.
A heading can help people to express the target more intuitively to others. There are three levels of headers in RStudio, they are distinguished by the number of comment signs or replaced by equal signs or dashes. With the help of the headers, the code could be more logical and readable.
-4 Packages:
We can find useful packages on the website https://cran.r-project.org/mirrors.html and download the package with the needed function. Once we decide on the package, we can also install it in the RStudio packages part or write a command in the console.
https://www.tidyverse.org/packages/
Core tidyverse
The definition of tidy data is that each column is a variable, and each row is an observation.
The core tidyverse includes the packages that you’re likely to use in everyday data analyses. As of tidyverse 1.3.0, the following packages are included in the core tidyverse:
ggplot2
ggplot2 is a system for declaratively creating graphics, based on The Grammar of Graphics. You provide the data, tell ggplot2 how to map variables to aesthetics, and what graphical primitives to use, and it takes care of the details.
dplyr
dplyr provides a grammar of data manipulation, providing a consistent set of verbs that solve the most common data manipulation challenges.
tidyr
tidyr provides a set of functions that help you get to tidy data. Tidy data is data with a consistent form: in brief, every variable goes in a column, and every column is a variable.
readr
readr provides a fast and friendly way to read rectangular data (like csv, tsv, and fwf). It is designed to flexibly parse many types of data found in the wild, while still cleanly failing when data unexpectedly changes.
purrr
purrr enhances R’s functional programming (FP) toolkit by providing a complete and consistent set of tools for working with functions and vectors. Once you master the basic concepts, purrr allows you to replace many for loops with code that is easier to write and more expressive.
tibble
tibble is a modern re-imagining of the data frame, keeping what time has proven to be effective, and throwing out what it has not. Tibbles are data. frames that are lazy and surly: they do less and complain more forcing you to confront problems earlier, typically leading to cleaner, more expressive code.
stringr
stringr provides a cohesive set of functions designed to make working with strings as easy as possible. It is built on top of stringi, which uses the ICU C library to provide fast, correct implementations of common string manipulations.

forcats
forcats provides a suite of useful tools that solve common problems with factors. R uses factors to handle categorical variables, variables that have a fixed and known set of possible values.
-5 Piped Commands with %>%:
Piped commands are usually used in the nested forms of composite functions. They are contained in the dplyr package in R to show a sequence of operations on a data frame. The sequence clearly shows which function is used in which data. The result generated in the previous line will be imported into a later operation.
-6 Import data:
R’s built-in datasets:
The datasets package is installed by default but not loaded, so we also need to run ‘library()’ command to load datasets. To get help on packages, we can enter ?datasets. Library(help=”dataset”) can get the list of datasets.
Explore datasets with pacman:
Pacman, the package manager, can be prepared by ‘if (!require(“pacman”)) install.packages(“pacman”)’ ‘pacman::p_load(datasets, pacman, rio, tidyverse)’
p_data() function is from pacman package, then the function inside can be filled by those packages above. (ggplot2, dplyr, tidyr, stringr, forcats). There are also many packages available such as carData, caret, cluster, DescTools, GGally, gnm,janeaustenr, Lahman, lava, lmtest, maps, MASS, mlbench,nlme, nycflights13, psych, qcc, quantreg, rpart, SixSigma, survival, titanic, vcd

Import XML data:
Data from the web are hierarchically structured data. XML files are commonly used to store those data, which stands for Extensible Markup Language. In RStudio, we can turn the nested structure into the kind of rectangular data frame.
To begin with, we can download some packages. XML2R is for working with XML data. For example, we can find more information on ergast.com. If we want to download the data from the web, it means the internet should be connected. df stands for data frame.
After a series of transformations, the nested structure is changed to a small data frame, which is more convenient to handle and more intuitive.
In extracting and combing data processes, we can extract data with the select( ) function or filter() and then print them out.
Import JSON data:
It is an older method that stands for JavaScript object notation. The approach to importing data is the same as the previous part of XML.
First, we need to contribute jsonlite with the command pacman::p_load(pacman, tidyverse, jsonlite) and then enter from JSON(). The approaches to locating data are the same as in the previous part.
Saving data in native R formats:
save(df, file = “data/StateData.rdata”)
Saving a file as a format of rdata is much smaller than the common form csv or xlsx. After loading the saved data file, we can choose base Rs’s load() or rio’s import() to read Rdata files.
‘import’ is a command from the ‘rio’ package. It doesn’t display immediately in the environment but shows in the console. We can also save this form of a file by renaming it.
