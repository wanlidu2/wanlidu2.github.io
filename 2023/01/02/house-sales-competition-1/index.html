You have some experience with R or Python and machine learning basics. This is a perfect competition for data science students who have completed an online course in machine learning and are looking to expand their skill set before trying a featured competition.
Housing Prices Competition
Category: Kaggle competition
Text:
This competition is for beginners and it is slightly easier than the ongoing competitions with rewards.
It requires competitors who have learned about R or python to gain some experience in solving this problem. The target is to predict the given data on house sales prices. Root Mean Squared Error is used for the accuracy of the predictions. In the final submission of predictions, each player should hand out a format with the Id and its corresponding SalePrice.
To help the starters better understand this competition, Kaggle also linked with relatively machine learning lessons and some R information.

The features of the dataset are nearly the same because of the fundamental of machine learning. It is that in order to estimate the performance of the machine learning model on new data, we can separate the data into two parts: the training set and the testing set. The training dataset is used to fit the machine learning model while the test dataset is used to evaluate the effect of the previously generated model. Though the data from these two parts are almost different, they should have the same features, and the same variables on the fitted model, to check the performance of the model. We can derive the model by using R or python code, which is also the main task of the competition.
The sample submission file is an example of the submission form which only has two columns: Id and SalePrice.
Problem Solving:
This competition is also a good application of knowledge about machine learning. In general, the original dataset contains some features we want but may miss some data as well. Facing real-world datasets, we need to master the skills of dealing with those problems, such as missing values, and categorical variables. We need to distinguish between numeric and alphabetic variables and then use a different method to classify and handle them.
In the advanced sections of machine learning, we have many tools to improve the performance of the model estimation results. By designing pipelines to compact the programming code and improve the quality, using cross-validation to improve the fitted model, building the XGBoost function to get a higher ranking in the game, and avoiding leakage, overfitting, or underfitting, we can improve our model in different side.
General logic:
Using python language
1- import the data:
Using pandas: read CSV command to import data; then use a model in the sklearn package. “train_test_split” command is to split one file into two parts and check the model by cross-validation. At this step, we can set different train sizes between 0 and 1, a numerical random state to divide the dataset in different ways.
In order to keep the useful features from the complete dataset, we can first set the features as what we want then copy the original files and limit the data in those columns. It is a useful way to reduce data while avoiding the influence of irrelevant factors on the model.
2- compare different models then keep the best model:
One of the most common models of the sklearn package is the Random Forest Model. The main idea of this regressor is by fitting a number of classifying decision trees on various subsamples of the dataset and using averages to improve the predictive accuracy while controlling over-fitting. Simply in this command, we can set different values of n_estimators (default=100), random_state, criterion (different function to measure the quality of a split e.g.: squared_error), max/min_depth/sample_split) … In the face of an unknown model, there are many parameters that we can decide on and change. In the beginning, we should try as many combinations of different initial settings as possible.
Recall the requirements of the competition, we need to use MAE to evaluate the accuracy of the model. We can import MAE from the sklearn.metrics. Since we create more than one model in the last step, it is easier to compare the result by defining a function that can be imported by model and parameters then printing out the absolute values and turning it to be readable and easy to compare.
After comparing the value of the function output, we can define the best model as the final model and take the same generate progress with the random forest regressor command to get the fitted value of the model.
3- Generate the output and submit
In the beginning, we define X and y from the original dataset, here we can use ‘model.fit(X,y)’ to fit the model and generate the test predictions by assigning the value equal to model.predict(X_test). The final step is to generate the output to a file by the ‘to csv’ command in pandas’ data frame package. For an unknown data set, it is relatively simple to get a model, but the difficulty lies in how to choose the method and adjust the parameters to obtain a more accurate model. The selection of models and the estimation of parameters will be the problems that every data analysis learner has to face.
