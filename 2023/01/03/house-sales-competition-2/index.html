Housing Prices Competition
Category: Kaggle competition
Text:
This article meanly introduces some important methods dealing with the real-world complicated dataset.
-1 Missing values
Not every column in the dataset has an exact value in the dataset. Those missing values show in the data as NaN which stands for not a number. Dealing with those missing values, the analysts can choose to drop those columns with missing values or get imputed values with some number generated by other samples. If we choose to add some imputation to fill the blank in the dataset, we can also add a new column remarking on the action of added imputed values.
First, we need to have a general understanding of the dataset. We can use ‘dataset.shape’ command to display the number of rows and columns. Besides ‘isnull’ command is specifically used to display missing values.
If we want to drop the columns with missing values, we can generate a new dataset with ‘.drop’ expression. In this step, col is short for columns in python language. Besides, in numpy and pandas, axis=0 stands for the actions to the horizontal axis while axis=1 refers to the vertical axis.
On the other hand, we can calculate the imputed value with the help of sklearn packages. One of the most simple ways is to enter the SimpleImputer command. There is another point that needs to be noticed. In the fitting process, we can use ‘fit_transform’ or ‘transform’. The former can finish both fits and transform: compute the mean and standard deviation for a given feature and also perform scaling using mean and standard deviation, the latter only works for transformation.
Same as before, we can calculate MAE from different approaches and choose the best method.
-2 Categorical variables
There are many non-numeric descriptive features in the common data set. Different from numerical characteristics which we can easily summarize by ‘describe()’ command. Generally speaking, the categorical variables have the following forms: words that indicate frequency, such as never, rarely, or every day, and nouns that describe characteristics, such as countries, brands, vehicles, etc.
For those objects, we want to compare and find the correlations among them which are the same as the numerical data.
One of the easiest ways to deal with them is to drop all the variables. However, removing useful information directly is a waste and also leads to inaccurate model estimation. As a result, this approach is not recommended.
Ordinal Encoding:
This method turns each unique value into a different integer. The size of the number also is defined logically. In order to solve the question, this integer reflects the relevance or importance. This programming method is usually used for turning an indisputable ranking into categories.
One-Hot Encoding
It creates new columns using a 0 or 1 to represent the absence or the presence of the categorical value. If the original data does not have some intrinsic ranking, it is a good way to transform it into nominal variables.
-3 Pipelines
In the common machine learning process, we use the train and test data and then generate an operationalizing model which could take a lot of time. Instead, pipelines are a useful way to not only compact your code and also organize the model.
There are many advantages to using pipelines. It can save time for repetitive work, improve the data-driven decision, and optimize strategies. For programmers, it provides cleaner code, fewer bugs, easier to produce, and also more options for model validation.
The sklearn packages provide us with some already-built functions in the pipeline approach. The same as the previous section, we can also call the column transformer, a simple imputer, one hot encoder function. After choosing the steps of the preprocessor and the model, we can fit the model and generate predictions and evaluate the model by calculating MAE.
To build a pipeline for the model, we can use the simple imputer to replace the missing values, call the random forest regressors to train a random forest model, then we can make a preliminary prediction. The number of trees in the random forest model is set by the n_estimators parameter, and the reproducibility is decided by random_state.
-4 Cross-Validation
Cross-validation is a useful way to measure the models’ performance. By cross-selecting the dataset, we can make the most use of it while avoiding over-fitting. If there is a large validation in the data, it displays less randomness which is harmful to the measure of model quality, so the result would be less reliable. After choosing the predictive variables to construct the model, we can use cross-validation to support the model.
The basic principle of cross-validation is that after dividing the data into training and validation parts, we change the role of each part to check the model’s quality. By repeating these steps, we can compare the result of the cross-validation and then get the best model.
The size of the dataset also helps analysts to decide whether to use cross-validation or not and how is the effect. For example, if the dataset is small, the calculation of extra computation is not very heavy, so it is suitable for this method. By contrast, a larger dataset is not guaranteed to save time and have a better result by using this measuring approach.
The cross-validation can also be called from the sklearn package. By comparing the score of cross-validation, with the help of MAE, we can measure the model quality and get a more convincing report.
In the corresponding exercise, we can calculate the average score of MAE by using cross-validation for the whole original dataset.
To construct a cross-validation evaluation model, we first need to define a function that can give scores to those different models. In this specific case, the error we want to be as small as possible. As a result, we can turn the error into a negative absolute error to fit the requirement in the scikit-learn convention, in which all metrics are evaluated as a higher number is better. The reason to take the average across value is that simply in one task we prefer to just compare one single measure of model quality. The next step is to test different parameter values. Setting while and for loop can make the process lean and intuitive. Then we can find out the best parameter value. As a matter of fact, the setting of n and the interval of the loop are equally important. For example, if we multiply i by 50, the interval is 50, and the result of n with best estimators is 200. However, the score of 205 is slightly smaller than 200. If we limit the calculation interval between 100 and 200, we won’t find out the value of 240 is better. How to set those parameters can not be ignored if we want to have a better solution.
-6 XGBoost
XGBoost can help us to build and optimize models with gradient boosting. Unlike the random forest model, XGBoost is an ensemble method combing the predictions of several models. It is initially generated with a single model, which prediction is also very simple. Then we can construct a cycle and add the content step by step. First, we use the current ensemble to generate predictions for each observation in the dataset and add more predictions from all models in the ensemble. Using these predictions we can have a loss function, which is used to fit a new model that later will be added to the ensemble. By repeating this, errors can be further reduced.
XGBoost stands for extreme gradient boosting, which is also an improvement in the performance and speed of gradient boosting. One approach to construct and fit the model is by using sklearn API for XGBoost. We will also learn more about tunable parameters. For example: n_estimators, early_stopping_rounds, learning_rate, n_jobs, etc. The most common parameter we might use is n_estimators. It defines how many times to go through the modeling cycle in the previous set. It is equal to the number of models in the ensemble. However, if we set the value too small or too large, it will cause the consequences of underfitting and overfitting. Therefore, it is necessary for us to learn more about how to avoid these mistakes.
